  <files>provider_agent/api_client.py</files>
  <action>
Create `provider_agent/api_client.py` implementing the data fetching layer with automatic fallback.

**Functions to implement (in order):**

1. **`_get_client() -> Client | None`**
   - Read `AMADEUS_CLIENT_ID` and `AMADEUS_CLIENT_SECRET` from environment variables (`os.environ.get()`)
   - If either is missing or empty → return `None` (per locked decision: missing credentials → skip API)
   - Otherwise, return `Client(client_id=..., client_secret=...)` — SDK defaults to sandbox/test hostname

2. **`_call_amadeus(client, destination, check_in, check_out, guests, max_price) -> list`**
   - Two-step Amadeus Hotel Search v3 flow (per research, v2 single-call is deprecated):
     - **Step 1:** `client.reference_data.locations.hotels.by_city.get(cityCode=city_code)` — get hotel IDs
     - **Step 2:** `client.shopping.hotel_offers_search.get(hotelIds=..., adults=..., checkInDate=..., checkOutDate=..., currency='EUR')` — get offers
   - City code mapping: maintain a `CITY_CODES` dict — `{"Berlin": "BER", "berlin": "BER"}`. For unknown cities, try `destination.upper()[:3]` as fallback.
   - Limit hotel IDs to first 20 from Step 1 (API limit is ~50, stay safe)
   - If Step 1 returns empty → raise `ValueError("No hotels found")`
   - Optionally include `priceRange=f"0-{int(max_price)}"` if max_price is provided (per research: may or may not be respected by sandbox)
   - Return `response.data` (the SDK parses JSON automatically)

3. **`_normalize_response(raw_data, check_in, check_out) -> list[dict]`**
   - Transform Amadeus nested response into flat dicts matching scorer format
   - Calculate `num_nights = max((check_out - check_in).days, 1)` to avoid division by zero
   - For each item in raw_data:
     - Extract `hotel_info = item.get("hotel", {})`
     - Extract `offers = item.get("offers", [])` — skip items with no offers
     - Take first offer: `offer = offers[0]`
     - Build normalized dict:
       ```python
       {
           "id": hotel_info.get("hotelId", "unknown"),
           "name": hotel_info.get("name", "Unknown Hotel"),
           "price_per_night": float(offer.get("price", {}).get("total", "0")) / num_nights,
           "rating": hotel_info.get("rating"),  # Often None in sandbox — scorer handles this
           "distance_km": 0.0,  # Not available from offers endpoint — scorer handles with neutral score
           "amenities": [],  # Not in offers response — scorer handles empty list
           "free_cancellation": any(
               p.get("type") == "FULL_REFUND"
               for p in offer.get("policies", {}).get("cancellations", [])
           ),
       }
       ```
   - Use `.get()` with fallback defaults for EVERY field — Amadeus sandbox often has missing data (per research Pitfall 1)
   - Return list of normalized hotel dicts

4. **`search_hotels(destination, check_in, check_out, guests, max_price=500.0) -> tuple[list[dict], bool]`**
   - Main entry point. Returns `(hotels, is_mock)` tuple.
   - **Per locked decisions:**
     - No retries — fail once, fall back immediately
     - 5-second timeout intention (if SDK doesn't support timeout natively, rely on try/except — for hackathon demo the sandbox is typically fast, per research Open Question 1)
     - Missing credentials → skip API entirely → mock data
     - ALL exceptions → mock data
   - Flow:
     ```
     client = _get_client()
     if client is None:
         return get_mock_hotels(), True

     try:
         raw = _call_amadeus(client, destination, check_in, check_out, guests, max_price)
         hotels = _normalize_response(raw, check_in, check_out)
         if not hotels:
             return get_mock_hotels(), True
         return hotels, False
     except Exception:
         return get_mock_hotels(), True
     ```
   - The broad `except Exception` is intentional (per locked decision: no distinction between error types)
