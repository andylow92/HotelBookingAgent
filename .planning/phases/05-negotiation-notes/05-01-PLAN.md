**Algorithm (from RESEARCH.md, follow exactly):**

1. **Guard clauses:**
   - `len(options) == 0` → return `""`
   - `len(options) == 1` → return `"Only one option matched your criteria: {name} at €{price:.0f}/night."`
   
2. **Identify roles by tag** (do NOT assume indices — read `.tag` from each option):
   - `best` = option with tag `"BEST_BALANCE"` (always exists, it's idx 0)
   - `cheapest` = option with tag `"CHEAPEST"` (may be None if only 2 results and tags overlap)
   - `highest_rated` = option with tag `"HIGHEST_RATED"` (may be None if <3 results)
   - Use a simple loop or dict comprehension: `{o.tag: o for o in options if o.tag}`

3. **Detect dominant weight** to determine emphasis order:
   - Build dict `{name: value}` for the 5 weight dimensions
   - Find the dimension with the highest weight value
   - This determines which comparison leads the note

4. **Compose the note using f-strings:**
   - **Lead sentence:** Describe the BEST_BALANCE option — name, price (€ whole euros /night), rating (1 decimal), distance (1 decimal km), free cancellation status
   - **Cheapest comparison** (if cheapest exists and is different hotel from best):
     - Price savings: `best.price_per_night - cheapest.price_per_night` (show as "€{diff:.0f}/night cheaper")
     - Tradeoffs: rating drop, distance change, cancellation loss — only mention if meaningfully different
   - **Highest-rated comparison** (if highest_rated exists):
     - Rating advantage: `highest_rated.rating - best.rating` (show as "rated {x:.1f} vs {y:.1f}")
     - Tradeoffs: price premium, distance change
   - When the dominant weight is NOT price, reorder sentences to lead with that dimension's comparisons

5. **Adaptive length:**
   - If all options have total_score spread < 0.05 (max - min): use short note "All three options are closely matched — {best.name} edges ahead slightly at €{price:.0f}/night."
   - Otherwise: full comparison note

6. **Formatting rules (LOCKED from CONTEXT.md):**
   - Currency: `€` prefix, whole euros for per-night (e.g., "€105/night")
   - Distances: one decimal km (e.g., "0.8km")
   - Ratings: one decimal out of 5 (e.g., "rated 4.4")
   - Hotel names: full name on first mention
   - Do NOT expose raw 0–1 scores — translate to concrete values
   - Tone: warm, professional travel advisor — not robotic, not casual

7. **Edge case handling:**
   - Identical prices → say "similarly priced" instead of "€0/night cheaper"
   - Both distance_km == 0.0 → omit distance comparison
   - When CHEAPEST == BEST_BALANCE (same hotel got both tags) → only compare against HIGHEST_RATED
